# Task: Contract Generation Templates

**Date**: 2026-01-12 **Status**: ✅ Completed **Target Repo**: `open-captable-protocol-daml`

## Goal

Replace programmatic string building in `generate-captable.ts` with a template-based approach using
placeholder files. This makes the generated contract structure visible and reviewable directly in
GitHub.

## Background

The current `generate-captable.ts` builds DAML code by concatenating strings in TypeScript:

```typescript
function generateProcessCreate(types: TypeDef[]): string {
  const cases = sortedTypes.map((t) => generateCreateCase(t)).join('\n');
  return `processCreate ctx maps createData = case createData of
${cases}`;
}
```

This approach has several drawbacks:

- **Hard to review**: The actual contract structure is buried in TypeScript string interpolation
- **Hard to understand at a glance**: Must read/run the generator to see what choices exist
- **Error-prone**: String building makes typos and formatting errors easy to miss
- **IDE support lost**: No DAML syntax highlighting or validation in template strings

## Current State

### What We Generate

`CapTable.daml` contains:

- 1 template (`CapTable`)
- 48+ OCF type maps (stakeholders, stock_classes, etc.)
- 1 batch choice (`UpdateCapTable`)
- Sum types (`OcfCreateData`, `OcfEditData`, `OcfObjectId`)
- Helper functions (`processCreate`, `processEdit`, `processDelete`, etc.)

### Current Generator Flow

```
captable-config.yaml → generate-captable.ts → CapTable.daml
                              ↓
                    OCF/*.daml files (type discovery)
```

## Proposed Solution

### Template Structure

Two types of templates:

- **Main template**: Contains all static DAML with `{{#each}}` blocks for per-type sections
- **Loop templates**: Snippets that repeat for each OCF type (used inside `{{#each}}` blocks)

```
scripts/codegen/templates/
├── CapTable.daml.template     # Main template (static + loops inline)
└── loops/
    ├── create-case.daml       # Case in processCreate (per type)
    ├── edit-case.daml         # Case in processEdit (per type)
    └── delete-case.daml       # Case in processDelete (per type)
```

### Main Template Example

The main template contains all static DAML inline. Per-type sections use `{{#each}}` with loop
templates:

```daml
module Fairmint.OpenCapTable.CapTable where

-- =============================================================================
-- CapTable Contract (GENERATED - DO NOT EDIT)
-- =============================================================================

import DA.Map (Map)
import qualified DA.Map as Map
import DA.List (sortOn)
import DA.Action (foldlA)

import Fairmint.OpenCapTable.Types (Context)
import Fairmint.OpenCapTable.Helpers (createMarker)
import Fairmint.OpenCapTable.OCF.Issuer (Issuer(..), IssuerOcfData)

-- OCF Types (per-type loop)
{{#each types}}
import {{module}} ({{name}}(..), {{data_type}})
{{/each}}

-- =============================================================================
-- Batch Operation Types
-- =============================================================================

-- Sum type constructors (per-type loop)
data OcfCreateData
{{#each types}}
  {{#if @first}}={{else}}|{{/if}} OcfCreate{{name}} {{data_type}}
{{/each}}
  deriving (Eq, Show)

-- ... rest of sum types inline ...

-- =============================================================================
-- Helper Functions
-- =============================================================================

processCreate : Context -> CapTableMaps -> OcfCreateData -> Update (CapTableMaps, Text)
processCreate ctx maps createData = case createData of
{{#each types}}
{{> create-case}}
{{/each}}

-- ... rest of helpers with their loops ...

-- =============================================================================
-- CapTable Template
-- =============================================================================

template CapTable
  with
    context: Context
    issuer: ContractId Issuer

    -- OCF maps (per-type loop)
{{#each types}}
    {{map_field}}: Map Text (ContractId {{name}})
{{/each}}

  where
    signatory context.issuer, context.system_operator

    -- EditIssuer and UpdateCapTable choices inline...
```

### Loop Template Example

`loops/create-case.daml` — renders once per OCF type inside `processCreate`:

```daml
      OcfCreate{{name}} d -> do
        assertMsg "{{name}} ID already exists" (Map.lookup d.id maps.{{map_field}} == None)
{{#each validations}}
        assertMsg ("{{error}}: " <> d.{{field}}) (Map.lookup d.{{field}} maps.{{map}} /= None)
{{/each}}
        cid <- create {{name}} with context = ctx, {{data_param}} = d
        let newMaps = maps with {{map_field}} = Map.insert d.id cid maps.{{map_field}}
        pure (newMaps, d.id)
```

### Template Engine: Handlebars

Use Handlebars for template rendering:

- Mature, well-supported, strong TypeScript support
- `{{#each}}` for loops, `{{> partial}}` for loop templates
- `npm install handlebars @types/handlebars`

function generate(): void {
  const config = loadConfig();
  const types = discoverTypes(config).sort((a, b) => a.name.localeCompare(b.name));

  const template = Handlebars.compile(fs.readFileSync('templates/CapTable.daml.template', 'utf-8'));
  fs.writeFileSync(OUTPUT_PATH, template({ types }));
}
