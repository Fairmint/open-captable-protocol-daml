# Task: OCF Schema Field Completeness

**Date**: 2026-01-15 **Status**: Complete ✅ **Target Repo**: `open-captable-protocol-daml`

## Goal

Fix two field-level discrepancies between OCF JSON schemas and DAML implementations to ensure full
schema compliance.

## Background

Verification of 56 OCF schemas against 48 DAML implementations revealed:

- **8 "missing" implementations are intentional** per
  [ADR-001](https://github.com/fairmint/canton/blob/main/docs/developer/adr/001-ocf-captable-on-canton.md):
  - `Financing` - Organizational metadata only, not required for cap table correctness
  - `PlanSecurity*` (7 types) - Deprecated OCF v1.x aliases (see policy below)

- **2 field-level issues** require fixes (detailed below)

## Comprehensive Schema Review (2026-01-15)

JSON schemas in `ocp-canton-sdk/generated/ocf-schema/` was completed.

2. **StockClass** - `initial_shares_authorized` is `Decimal` but should be union type

## Policy: No Deprecated Types or Fields

**We do not implement deprecated OCF types or fields.** This keeps the codebase clean and avoids
maintaining legacy compatibility shims.

## Implementation Plan

### 1. Add `OcfOptionGrantType` Enum

Add new enum type to shared types.

**Files to modify:**

- `OpenCapTable-v29/daml/Fairmint/OpenCapTable/Types.daml` — Add `OcfOptionGrantType` enum

```daml
-- Enum - Option Type
-- If the plan security is an option, what kind?
-- OCF: https://raw.githubusercontent.com/Open-Cap-Table-Coalition/Open-Cap-Format-OCF/main/schema/enums/OptionType.schema.json
data OcfOptionGrantType = OcfOptionNSO | OcfOptionISO | OcfOptionINTL
  deriving (Eq, Show)
```

### 2. Update EquityCompensationIssuance

Add optional `option_grant_type` field.

**Files to modify:**

- `OpenCapTable-v29/daml/Fairmint/OpenCapTable/OCF/EquityCompensationIssuance.daml` — Add field and
  import

**Schema reference**

### 3. Move `OcfInitialSharesAuthorized` to Shared Types

Currently defined in `Issuer.daml`, needs to be shared for reuse by `StockClass`.

**Files to modify:**

- `OpenCapTable-v29/daml/Fairmint/OpenCapTable/Types.daml` — Add types
- `OpenCapTable-v29/daml/Fairmint/OpenCapTable/OCF/Issuer.daml` — Import from Types instead

Move these types:

```daml
data OcfAuthorizedShares =
    OcfAuthorizedSharesNotApplicable
  | OcfAuthorizedSharesUnlimited
  deriving (Eq, Show)

data OcfInitialSharesAuthorized =
    OcfInitialSharesNumeric Decimal
  | OcfInitialSharesEnum OcfAuthorizedShares
  deriving (Eq, Show)
```

### 4. Update StockClass

Change `initial_shares_authorized` from `Decimal` to union type.

**Files to modify:**

- `OpenCapTable-v29/daml/Fairmint/OpenCapTable/OCF/StockClass.daml` — Change type and validation

**Current:**

```daml
initial_shares_authorized: Decimal
```

**After:**

```daml
initial_shares_authorized: OcfInitialSharesAuthorized
```

**Schema reference**
([StockClass.schema.json](../../../ocp-canton-sdk/generated/ocf-schema/StockClass.schema.json)):

```json
"initial_shares_authorized": {
  "oneOf": [
    { "enum": ["NOT APPLICABLE", "UNLIMITED"] },
    { "type": "string", "pattern": "^[+-]?[0-9]+(\\.[0-9]{1,10})?$" }
  ]
}
```

## Migration Notes

**Breaking Change**: `StockClass.initial_shares_authorized` type changes from `Decimal` to sum type.

- **Existing contracts**: Will need migration if reading directly as `Decimal`
- **SDK impact**: `ocp-canton-sdk` may need type conversion updates
- **Mitigation**: Numeric values remain valid via `OcfInitialSharesNumeric` constructor
