# CI Backwards Compatibility Check

**Date:** 2026-01-16 **Status:** Open

## Summary

Add CI validation to ensure DAML package changes are backwards compatible. The CI should fail if
changes would break upgrade compatibility, forcing developers to explicitly bump the major version
when introducing breaking changes.

## Background

DAML has built-in upgrade validation via `dpm upgrade-check` that can verify if a new DAR is
compatible with a previous version. This check ensures:

1. **Compiler checks** - Template/interface definitions remain compatible
2. **Participant checks** - Runtime upgrade paths are valid

## Implementation

### Upgrade Check Script

Create `scripts/check-upgrade-compatibility.ts` that:

1. For each package with a backed-up DAR in `dars/`:
   - Build the current package
   - Run `dpm upgrade-check --both <previous-dar> <current-dar>`
   - **Pass** if upgrade check succeeds (backwards compatible)
   - **Pass** if package is a new major version (e.g., v28 → v29)
   - **Fail** if upgrade check fails and version is unchanged

2. Detect major version changes by comparing:
   - Current package name (e.g., `OpenCapTable-v29`)
   - Most recent backed-up version (e.g., `OpenCapTable-v28`)

### CI Integration

Add step to `.github/workflows/ci.yml`:

```yaml
- name: Check backwards compatibility
  run: npm run check-upgrade-compat
```

### NPM Script

Add to `package.json`:

```json
"check-upgrade-compat": "tsx scripts/check-upgrade-compatibility.ts"
```

## Success Criteria

- [ ] `npm run check-upgrade-compat` script validates all packages
- [ ] CI fails if non-backwards-compatible changes without major version bump
- [ ] CI passes for new major versions (breaking changes are expected)
- [ ] CI passes for backwards-compatible changes
- [ ] Document the upgrade workflow in `UPGRADE_GUIDE.md`

## Example Scenarios

| Scenario                                           | Expected Result |
| -------------------------------------------------- | --------------- |
| Minor change to OpenCapTable-v29 (compatible)      | ✅ CI passes    |
| Breaking change to OpenCapTable-v29 (incompatible) | ❌ CI fails     |
| Breaking change + bump to OpenCapTable-v30         | ✅ CI passes    |
| New package with no previous DAR                   | ✅ CI passes    |

## Technical Details

### dpm upgrade-check Usage

```bash
# Check if new-dar is a valid upgrade from old-dar
dpm upgrade-check --both <old-dar> <new-dar>

# Options:
#   --compiler     Run compiler upgrade checks only
#   --participant  Run participant upgrade checks only
#   --both         Run both checks (recommended)
```

### Exit Codes

- `0` - Upgrade is valid
- `1` - Upgrade validation failed
