# Task: Package Upgrade Directory Generation

**Date**: 2026-01-15 **Status**: Open **Target Repo**: `open-captable-protocol-daml`

## Goal

Eliminate verbose diffs from major version upgrades by generating versioned package directories at
build time instead of committing them directly to git.

## Problem

Currently, major version upgrades rename entire directories (e.g., `OpenCapTable-v27/` →
`OpenCapTable-v28/`). This causes:

- **Verbose diffs**: Every file shows as deleted and re-added
- **Hard to review**: Actual code changes are buried in path renames
- **Import churn**: All `daml.yaml` dependencies and imports reference the versioned folder name

Example of a typical major upgrade diff:

```
deleted:   OpenCapTable-v27/daml/Fairmint/OpenCapTable/CapTable.daml
deleted:   OpenCapTable-v27/daml/Fairmint/OpenCapTable/Types.daml
... (50+ files deleted)
added:     OpenCapTable-v28/daml/Fairmint/OpenCapTable/CapTable.daml
added:     OpenCapTable-v28/daml/Fairmint/OpenCapTable/Types.daml
... (50+ files added)
```

## Proposed Solution

Keep **stable source directory names** in git (e.g., `OpenCapTable/` without version suffix) and
generate the versioned directories at build time. The version lives only in `daml.yaml`.

### Directory Structure

**Before (current):**

```
/workspace/
  OpenCapTable-v28/      # Committed - version in folder name
    daml.yaml            # name: OpenCapTable-v28, version: 0.0.1
    daml/...
  multi-package.yaml     # references: OpenCapTable-v28
```

**After (proposed):**

```
/workspace/
  OpenCapTable/          # Committed - stable name, no version suffix
    daml.yaml            # name: OpenCapTable-v28, version: 0.0.1 (version HERE)
    daml/...

  .build/                # Gitignored - generated at build time
    OpenCapTable-v28/    # Copied + transformed from OpenCapTable/
      daml.yaml
      daml/...

  multi-package.yaml     # Auto-generated to .build/multi-package.yaml
```

### How It Works

1. **Source directories** have stable names (`OpenCapTable/`, `Shared/`, etc.)
2. **Version info** lives in each package's `daml.yaml` (package name + version)
3. **Build script** copies source → `.build/<package-name>/` (where `package-name` comes from
   `daml.yaml`)
4. **dpm builds** from `.build/` directory
5. **Upgrade script** only updates `daml.yaml` — no folder renames

### Version Upgrade Flow

**Major upgrade** (v27 → v28):

```bash
npm run upgrade-package -- --package OpenCapTable --type major
```

Changes only `OpenCapTable/daml.yaml`:

```yaml
name: OpenCapTable-v27  →  name: OpenCapTable-v28
version: 0.0.3          →  version: 0.0.1
```

**Diff is clean:**

```diff
- name: OpenCapTable-v27
+ name: OpenCapTable-v28
- version: 0.0.3
+ version: 0.0.1
```

## Technical Details

### `scripts/prepare-build.ts`

```typescript
/**
 * Prepares the .build directory for dpm compilation. Copies source packages to versioned
 * directories based on daml.yaml names.
 */

import * as fs from 'fs';
import * as path from 'path';
import * as yaml from 'yaml';

const ROOT = process.cwd();
const BUILD_DIR = path.join(ROOT, '.build');
const SOURCE_MANIFEST = path.join(ROOT, 'multi-package.yaml');

interface DamlYaml {
  name: string;
  version: string;
}

function readDamlYaml(packageDir: string): DamlYaml {
  const yamlPath = path.join(packageDir, 'daml.yaml');
  return yaml.parse(fs.readFileSync(yamlPath, 'utf-8'));
}

function copyDir(src: string, dest: string): void {
  fs.mkdirSync(dest, { recursive: true });
  for (const entry of fs.readdirSync(src, { withFileTypes: true })) {
    const srcPath = path.join(src, entry.name);
    const destPath = path.join(dest, entry.name);
    if (entry.isDirectory()) {
      copyDir(srcPath, destPath);
    } else {
      fs.copyFileSync(srcPath, destPath);
    }
  }
}

function main(): void {
  // Clean and create build directory
  if (fs.existsSync(BUILD_DIR)) {
    fs.rmSync(BUILD_DIR, { recursive: true });
  }
  fs.mkdirSync(BUILD_DIR);

  // Read source package list
  const manifest = yaml.parse(fs.readFileSync(SOURCE_MANIFEST, 'utf-8'));
  const buildPackages: string[] = [];

  for (const sourceDir of manifest.packages) {
    const sourcePath = path.join(ROOT, sourceDir);
    const damlYaml = readDamlYaml(sourcePath);

    // Target directory uses the name from daml.yaml (includes version)
    const targetDir = damlYaml.name;
    const targetPath = path.join(BUILD_DIR, targetDir);

    console.log(`${sourceDir}/ → .build/${targetDir}/`);
    copyDir(sourcePath, targetPath);
    buildPackages.push(targetDir);
  }

  // Generate build multi-package.yaml
  const buildManifest = { packages: buildPackages };
  fs.writeFileSync(path.join(BUILD_DIR, 'multi-package.yaml'), yaml.stringify(buildManifest));

  console.log(`\nGenerated .build/multi-package.yaml with ${buildPackages.length} packages`);
}

main();
```

## Edge Cases

### Codegen Paths

`generate-captable.ts` currently hardcodes `OpenCapTable-v28/`:

```typescript
const OPENCAPTABLE_DIR = path.join(REPO_ROOT, 'OpenCapTable-v28/daml/Fairmint/OpenCapTable');
```

Change to:

```typescript
const OPENCAPTABLE_DIR = path.join(REPO_ROOT, 'OpenCapTable/daml/Fairmint/OpenCapTable');
```

### Import Statements in DAML

DAML imports use module paths, not file paths:

```daml
import Fairmint.OpenCapTable.OCF.Stakeholder
```

These are **not affected** by directory renames — only the folder structure matters for `dpm`.

### Cross-Package Dependencies

Dependencies in `daml.yaml` reference package names (with version):

```yaml
dependencies:
  - daml-prim
  - daml-stdlib
  - ../Shared
```

These become build-relative paths in `.build/`:

```yaml
dependencies:
  - daml-prim
  - daml-stdlib
  - ../Shared # Works because Shared is also in .build/
```

Wait, `Shared` doesn't have a version suffix. We need to handle this:

- Packages without version suffixes: source name = build name
- Packages with version suffixes: source name (stable) ≠ build name (versioned)

The `prepare-build.ts` script handles this by reading `daml.yaml` to determine the target name.
