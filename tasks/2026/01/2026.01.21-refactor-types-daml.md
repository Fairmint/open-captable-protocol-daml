# Refactor Types.daml

Status: Open Created: 2026-01-21 Last updated: 2026-01-21

## Context

`Types.daml` in `OpenCapTable` has grown to over 1000 lines. It contains a mix of common types, OCF
object definitions, conversion mechanisms, and validation logic.

## Goals

- Split `Types.daml` into smaller, more focused modules.
- Improve maintainability and readability.
- Standardize validation logic.

## Proposed Modules

- `Fairmint.OpenCapTable.Types.Common` (Basic types like Email, Phone, Address)
- `Fairmint.OpenCapTable.Types.Enums` (Simple enums)
- `Fairmint.OpenCapTable.Types.Conversion` (Conversion mechanisms and rights)
- `Fairmint.OpenCapTable.Types.Financial` (Monetary, Interest, Vesting)

## Success Criteria

- [ ] `Types.daml` is split into submodules.
- [ ] `Types.daml` re-exports the submodules for backward compatibility (if possible/desired) or
      imports are updated.
- [ ] All tests pass.
- [ ] Validation logic is consistent.
