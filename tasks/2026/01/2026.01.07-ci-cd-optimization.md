# Task: CI/CD Pipeline Optimization

**Date**: 2026-01-07
**Status**: Completed ✅
**Target Repo**: `open-captable-protocol-daml`

## Goal

Optimize the CI/CD pipeline to reduce build times, add caching, and improve reliability. Current builds take longer than necessary due to lack of caching and redundant operations.

## Background

The repository has two main workflows:
- `ci.yml` - Runs on all pushes, builds and tests DAML packages
- `publish.yml` - Runs on main branch, publishes to NPM

Both workflows lack optimization strategies that could significantly reduce build times.

## Current State

### Issues Identified

1. **No dependency caching**
   - npm dependencies reinstalled every run
   - DAML SDK downloaded fresh every build
   - No caching of `.daml` build artifacts

2. **DAML SDK installation fragility**
   - Manual curl-based installation with retry logic
   - No version pinning beyond what's in `daml.yaml`
   - Network failures can cause flaky builds

3. **Inconsistent Node versions**
   - CI uses Node 20, Publish uses Node 18
   - Should be consistent across workflows

4. **No build matrix**
   - Not testing against multiple DAML SDK versions
   - Could catch compatibility issues earlier

5. **Missing workflow optimizations**
   - No concurrency controls (multiple builds can run for same branch)
   - No job dependencies to fail fast

## Proposed Improvements

### Phase 1: Add Caching

```yaml
# Add npm caching
- name: Cache npm dependencies
  uses: actions/cache@v4
  with:
    path: ~/.npm
    key: ${{ runner.os }}-npm-${{ hashFiles('**/package-lock.json') }}
    restore-keys: |
      ${{ runner.os }}-npm-

# Add DAML SDK caching
- name: Cache DAML SDK
  uses: actions/cache@v4
  with:
    path: ~/.daml
    key: ${{ runner.os }}-daml-${{ env.SDK_VERSION }}
    restore-keys: |
      ${{ runner.os }}-daml-

# Add DAML build artifact caching
- name: Cache DAML build artifacts
  uses: actions/cache@v4
  with:
    path: |
      **/.daml
      generated/
    key: ${{ runner.os }}-daml-build-${{ hashFiles('**/*.daml', '**/daml.yaml') }}
```

### Phase 2: Version Consistency

- [ ] Standardize on Node 20 across all workflows
- [ ] Extract DAML SDK version to workflow variable
- [ ] Consider using official DAML action if available

### Phase 3: Concurrency Controls

```yaml
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true
```

### Phase 4: Fail-Fast Improvements

- [ ] Split build and test into separate jobs
- [ ] Run lint checks in parallel with build
- [ ] Only run publish after all checks pass

### Phase 5: Build Matrix (Optional)

```yaml
strategy:
  matrix:
    daml-version: ['3.3.0-snapshot.20250603.0']  # Add more versions as needed
```
