# Task: Standardize Validator Function Naming

**Date**: 2026-01-21 **Status**: Complete ✅ **Target Repo**: `open-captable-protocol-daml`

## Goal

Standardize all validator function names to follow a consistent pattern: `validate<TypeName>`.

## Problem

Current validator naming is inconsistent across the codebase:

| Pattern                 | Example                          | Count      |
| ----------------------- | -------------------------------- | ---------- |
| `validate*TxData`       | `validateOcfStockTransferTxData` | 35 files   |
| `validate*Data`         | `validateOcfStockIssuanceData`   | 11 files   |
| `validate*` (no suffix) | `validateOcfVesting`             | Types.daml |

**Examples of inconsistency:**

```daml
-- Transaction files use "TxData" suffix
validateOcfStockTransferTxData : StockTransferOcfData -> Bool
validateOcfConvertibleIssuanceTxData : ConvertibleIssuanceOcfData -> Bool

-- But some transaction files use just "Data"
validateOcfStockIssuanceData : StockIssuanceOcfData -> Bool  -- No "Tx"
validateOcfEquityCompensationExerciseData : EquityCompensationExerciseOcfData -> Bool

-- Types.daml uses no suffix for embedded types
validateOcfVesting : OcfVesting -> Bool
validateOcfMonetary : OcfMonetary -> Bool
```

This inconsistency makes it hard to:

- Predict function names when writing code
- Find validators with search
- Maintain consistency in new code

## Proposed Solution

Standardize on: **`validate<TypeName>`** where `<TypeName>` matches the data type name.

| Type                   | Validator Name                 |
| ---------------------- | ------------------------------ |
| `StockTransferOcfData` | `validateStockTransferOcfData` |
| `StockIssuanceOcfData` | `validateStockIssuanceOcfData` |
| `OcfVesting`           | `validateOcfVesting`           |
| `OcfMonetary`          | `validateOcfMonetary`          |

**Rule:** The validator name should be `validate` + the exact type name it validates.

### Alternative Considered

Could use `validateOcf*Data` consistently for all OCF data types, but this:

- Doesn't match embedded type names (`OcfVesting` vs `VestingOcfData`)
- Adds redundant "Ocf" prefix when type already has it
- Still requires deciding between "TxData" vs "Data"

## Implementation Plan

### Phase 1: Rename Validators in OCF Files

For each of the 35 affected files:

1. Rename the function from `validateOcf*TxData` to `validate*OcfData`
2. Update the `ensure` clause to use new name
3. Update any internal calls to the validator

**Example change in `StockTransfer.daml`:**

```daml
-- Before
ensure
  validateOcfStockTransferTxData transfer_data

validateOcfStockTransferTxData : StockTransferOcfData -> Bool
validateOcfStockTransferTxData d = ...

-- After
ensure
  validateStockTransferOcfData transfer_data

validateStockTransferOcfData : StockTransferOcfData -> Bool
validateStockTransferOcfData d = ...
```

## Automation Approach

This refactoring can be partially automated with a script:

```bash
# For each file, rename function and update references
for file in OpenCapTable-v30/daml/Fairmint/OpenCapTable/OCF/*.daml; do
  # Replace validateOcf*TxData with validate*OcfData
  sed -i 's/validateOcf\([A-Za-z]*\)TxData/validate\1OcfData/g' "$file"
done
```

However, manual review is recommended to ensure:

- The type signature still matches
- The `ensure` clause is updated
- No edge cases are missed
