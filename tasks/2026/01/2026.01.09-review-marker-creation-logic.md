# Task: Review Marker Creation Logic for CapTable Interactions

**Date**: 2026-01-09
**Status**: Open
**Target Repo**: `open-captable-protocol-daml`

## Goal

Review and fix marker creation logic so that markers are only created when calling the batch `UpdateCapTable` function, not on individual OCP contract choices.

## Background

Currently, the code generator (`scripts/codegen/generate-captable.ts`) adds `createMarker context` calls to every individual choice:
- `CreateXxx` (e.g., `CreateStakeholder`, `CreateStockClass`)
- `EditXxx` (e.g., `EditStakeholder`, `EditIssuer`)
- `DeleteXxx` (e.g., `DeleteStakeholder`)

This is incorrect. Markers should **only** be created when calling the batch `UpdateCapTable` choice. For individual OCP contract interactions, the subscription and rewards should not be affected.

## Current State (Incorrect)

Each individual choice creates a marker:

```daml
choice CreateStakeholder : ContractId CapTable
  with
    stakeholder: StakeholderOcfData
  controller context.issuer
  do
    -- validations...
    _ <- createMarker context  -- ❌ Should NOT create marker here
    -- create contract...
```

## Desired State (Correct)

Individual choices should NOT create markers:

```daml
choice CreateStakeholder : ContractId CapTable
  with
    stakeholder: StakeholderOcfData
  controller context.issuer
  do
    -- validations...
    -- NO createMarker call here ✅
    -- create contract...
```

Only the batch `UpdateCapTable` choice should create markers:

```daml
choice UpdateCapTable : UpdateCapTableResult
  with
    creates: [OcfCreateData]
    edits: [OcfEditData]
    deletes: [OcfObjectId]
  controller context.issuer
  do
    -- process creates, edits, deletes...
    _ <- createMarker context  -- ✅ Marker created once per batch
    -- return result...
```

## Code Changes

### `scripts/codegen/generate-captable.ts`

Remove `createMarker` from individual choice generators:

```typescript
function generateCreateChoice(t: TypeDef): string {
  const validations = generateValidations(t.validations, t.data_param, "        ");
  const validationBlock = validations ? `\n${validations}\n` : "";

  return `    choice Create${t.name} : ContractId CapTable
      with
        ${t.data_param}: ${t.data_type}
      controller context.issuer
      do
        assertMsg "${t.name} ID already exists" (Map.lookup ${t.data_param}.id ${t.map_field} == None)
${validationBlock}
        // REMOVED: _ <- createMarker context

        new_cid <- create ${t.name} with
          context = context
          ..

        create this with ${t.map_field} = Map.insert ${t.data_param}.id new_cid ${t.map_field}`;
}
```

Similar changes for `generateEditChoice()` and `generateDeleteChoice()`.

Also remove the import if `createMarker` is no longer used:

```typescript
// Remove from generated output if no longer needed:
// import Fairmint.OpenCapTable.Helpers (createMarker)
```
