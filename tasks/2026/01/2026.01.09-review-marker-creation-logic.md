# Task: Review Marker Creation Logic for CapTable Interactions

**Date**: 2026-01-09
**Status**: ✅ Completed
**Target Repo**: `open-captable-protocol-daml`

## Goal

Review and fix marker creation logic so that markers are only created when calling the batch `UpdateCapTable` function, not on individual OCP contract choices.

## Summary of Changes

This task has been completed as part of removing `featured_app_right` from `OcpFactory` and `Context`. The following changes were made:

1. **Removed `featured_app_right` from `OcpFactory`** - The factory no longer holds a FeaturedAppRight reference
2. **Removed `featured_app_right` from `Context`** - The context type is now simpler with just `issuer` and `system_operator`
3. **Removed `createMarker` from individual choices** - All `CreateXxx`, `EditXxx`, and `DeleteXxx` choices no longer create markers
4. **Removed `createMarker` from `EditIssuer`** - The hardcoded EditIssuer choice no longer creates markers
5. **Removed `createMarker` from `CreateCapTable`** - IssuerAuthorization no longer creates markers when creating a CapTable
6. **Added `UpdateCapTable` with `AppRewardsConfig`** - The batch choice is the ONLY place where activity markers are created
