# Task: Code Generation Enhancements

**Date**: 2026-01-07
**Status**: Open
**Target Repo**: `open-captable-protocol-daml`

## Goal

Improve the `CapTable.daml` code generation system to be more robust, maintainable, and extensible for future requirements.

## Background

The repository uses code generation (`scripts/codegen/generate-captable.ts`) to create `CapTable.daml` from OCF type definitions. This approach reduces boilerplate but has room for improvement.

## Current State

### What's Good
- ✅ Automatic discovery of OCF types from `.daml` files
- ✅ Consistent choice generation (Create/Edit/Delete)
- ✅ YAML config for reference validations
- ✅ Clear separation of concerns

### Improvement Opportunities

1. **No Generator Tests**
   - Code generation logic has no unit tests
   - Regressions can slip in unnoticed
   - Output verification relies on manual review

2. **Limited Error Handling**
   - Parser failures show warnings but continue
   - Silent failures possible for edge cases
   - No validation of generated output

3. **Config Could Be More Expressive**
   - Only validation rules in config
   - Could support custom choice generation
   - Could support lifecycle-based ordering

4. **No Incremental Generation**
   - Full regeneration every build
   - Could be slower as types grow
   - No caching of parsed results

5. **Generated Code Not Idiomatic**
   - Some formatting inconsistencies
   - Comments could be more helpful
   - No link to source type in generated choices

### Phase 1: Add Generator Tests

```typescript
// scripts/codegen/__tests__/generate-captable.test.ts
describe('generate-captable', () => {
  describe('toSnakeCase', () => {
    it('should convert PascalCase to snake_case', () => {
      expect(toSnakeCase('StockIssuance')).toBe('stock_issuance');
    });
  });
  
  describe('parseTypeInfo', () => {
    it('should extract data type from DAML file', () => {
      const result = parseTypeInfo('/path/to/StockIssuance.daml', 'StockIssuance');
      expect(result?.dataType).toBe('StockIssuanceOcfData');
    });
  });
  
  describe('discoverTypes', () => {
    it('should find all OCF types except Issuer', () => {
      const types = discoverTypes(mockConfig);
      expect(types.map(t => t.name)).not.toContain('Issuer');
    });
  });
});
```
