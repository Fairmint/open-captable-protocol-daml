# Task: Test Coverage and Quality Improvements

**Date**: 2026-01-07
**Status**: Open
**Target Repo**: `open-captable-protocol-daml`

## Goal

Establish formal test coverage thresholds, improve test patterns, and add missing test scenarios to ensure code quality and prevent regressions.

## Background

The repository has comprehensive DAML tests (~50+ test files), but lacks:
- Formal coverage thresholds/enforcement
- TypeScript script tests
- Integration tests
- Negative test cases (testing error conditions)

## Current State

### What's Good
- ✅ Extensive DAML script tests for all OCF object types
- ✅ Test coverage flag enabled (`daml test --show-coverage`)
- ✅ Coverage exclusions configured properly in `daml.yaml`
- ✅ Consistent test naming conventions (`TestXxx.daml`)
- ✅ Setup helpers centralized in `Setup.daml`

### Gaps Identified

1. **No Coverage Thresholds**
   - Tests run but no minimum coverage enforced
   - Coverage reports not saved/tracked over time

2. **No TypeScript Tests**
   - Scripts in `scripts/` have no tests
   - `test-imports.ts` only checks import resolution

3. **Missing Negative Tests**
   - Most tests only verify happy path
   - Few tests verify validation failures/error messages

4. **No Integration Tests**
   - No tests verifying end-to-end workflows
   - No tests for factory → cap table → transaction flows

5. **PaymentStream Tests Incomplete**
   - `TestUtils.daml` in PaymentStreams is sparse
   - Some complex scenarios may be missing coverage

## Implementation Plan

### Phase 1: Coverage Enforcement

- [ ] Add coverage threshold to CI pipeline
- [ ] Create coverage badge for README
- [ ] Archive coverage reports as CI artifacts

```yaml
# In ci.yml
- name: Test DAML with coverage
  run: |
    npm run test 2>&1 | tee coverage.txt
    
- name: Check coverage threshold
  run: |
    # Parse coverage and fail if below threshold
    coverage=$(grep "Overall coverage" coverage.txt | ...)
    if [ "$coverage" -lt "80" ]; then
      echo "Coverage $coverage% is below 80% threshold"
      exit 1
    fi
```

## Test Patterns to Document

### DAML Test Naming
```
test<Type>_<Action>[_<Variant>]
testCreateStockIssuance           # Happy path
testCreateStockIssuance_DuplicateFails  # Negative
testCreateStockIssuance_AllOptionals    # Variant
```

### Test Structure
```daml
testXxx = script do
  -- 1. Setup
  TestOcp{...} <- setupTestOcp
  
  -- 2. Create prerequisites
  capTableCid <- submit issuer do ...
  
  -- 3. Execute action under test
  result <- submit issuer do ...
  
  -- 4. Verify result (implicit in DAML - test passes if no errors)
  pure ()
```
