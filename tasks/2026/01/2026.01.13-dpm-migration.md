# Task: Migrate from daml to dpm CLI

**Date**: 2026-01-13 **Status**: Open 🔄 **Target Repo**: `open-captable-protocol-daml`

## Goal

Migrate from the deprecated `daml` CLI (Daml Assistant) to `dpm` (Digital Asset Package Manager) as
the primary build tool for DAML packages.

## Background

Per [Canton 3.4 Release Notes](https://blog.digitalasset.com/developers/release-notes/canton-3.4-release-notes-for-splice-0.5.0):

> From version 3.4.x onwards, dpm is the preferred CLI tool for a local development experience...
> As a result of the introduction of dpm, the Daml Assistant (daml) is deprecated as of version 3.4,
> but still can be used to work with 3.3 and 3.4 daml projects.

The `daml` assistant will be **removed in Canton 3.5**, so migration is required before upgrading.

## `@daml/ledger` Deprecation

### Current Situation

The repo uses `@daml/ledger` (v2.10.3) as a dependency:

```json
{
  "dependencies": {
    "@daml/ledger": "2.10.3",
    "@daml/types": "3.4.10"
  }
}
```

Per release notes: _"The deprecated @daml/ledger and @daml/react typescript libraries are no longer
distributed with Canton 3.4."_

### Impact Assessment

1. **Generated JS bindings** - The `daml codegen js` / `dpm codegen-js` output imports from
   `@daml/ledger` and `@daml/types`
2. **bundle-dependencies.ts** - Creates stubs that import from `@daml/ledger`
3. **Downstream SDK consumers** - `ocp-canton-sdk` uses the generated types

### Mitigation Options

1. **Continue using 2.x versions** - The `@daml/ledger@2.10.3` package should still be available on
   npm as it was published before 3.4. Monitor for security issues.

2. **Investigate `dpm codegen-alpha-typescript`** - The new alpha TypeScript codegen may generate
   code without `@daml/ledger` dependency. Test if viable.

3. **Use JSON Ledger API directly** - Replace `@daml/ledger` with direct HTTP calls to the JSON
   Ledger API v2. The `canton-node-sdk` already uses this approach.

4. **Create custom type definitions** - If the generated types are minimal, we could maintain our
   own type definitions without `@daml/ledger`.

### Recommended Path

For now, keep `@daml/ledger@2.10.3` as it still works. Plan to investigate
`dpm codegen-alpha-typescript` as part of a future Canton 3.5 upgrade.
