# Task: Add App Rewards to Batch UpdateCapTable Choice

**Date**: 2026-01-08
**Status**: Open
**Target Repo**: `open-captable-protocol-daml`
**Related PR**: [#94 - feat: add batch UpdateCapTable choice](https://github.com/Fairmint/open-captable-protocol-daml/pull/94)

## Goal

Extend the batch `UpdateCapTable` choice (from PR #94) to accept featured app reward information, enabling the caller to specify how many coupons should be created per batch call along with the FeaturedAppRight contract ID.

## Background

The current `UpdateCapTable` choice in PR #94 uses the `featured_app_right` from the `Context` stored on the `CapTable` contract to create activity markers. This task adds flexibility by allowing the caller to pass this information per-call, supporting use cases where:

1. Different batch operations need different coupon counts
2. The FeaturedAppRight may change between calls
3. Callers want explicit control over reward creation

## Current State (PR #94)

```daml
choice UpdateCapTable : UpdateCapTableResult
  with
    creates: [OcfCreateData]
    edits: [OcfEditData]
    deletes: [OcfObjectId]
  controller context.issuer
```

## Proposed Change

Add an optional `appRewards` parameter:

```daml
-- | App reward configuration for batch operations
data AppRewardsConfig = AppRewardsConfig with
    couponCount: Int
    featuredAppRight: ContractId FeaturedAppRight
  deriving (Eq, Show)

choice UpdateCapTable : UpdateCapTableResult
  with
    creates: [OcfCreateData]
    edits: [OcfEditData]
    deletes: [OcfObjectId]
    appRewards: Optional AppRewardsConfig  -- NEW
  controller context.issuer
```

## Code Changes

### `CapTable.daml` (generated)

```daml
import Splice.Api.FeaturedAppRightV1 (FeaturedAppRight(..))

-- | App reward configuration for batch operations
data AppRewardsConfig = AppRewardsConfig with
    couponCount: Int
    featuredAppRight: ContractId FeaturedAppRight
  deriving (Eq, Show)

choice UpdateCapTable : UpdateCapTableResult
  with
    creates: [OcfCreateData]
    edits: [OcfEditData]
    deletes: [OcfObjectId]
    appRewards: Optional AppRewardsConfig
  controller context.issuer
  do
    -- Process creates, edits, deletes...
    
    -- Create activity markers
    case appRewards of
      Some config -> do
        -- Create couponCount markers using provided featuredAppRight
        replicateA_ config.couponCount $ do
          createActivityMarker beneficiaries config.featuredAppRight
      None -> do
        -- Fall back to single marker using context
        createMarker context
    
    -- Return result...
```

### `generate-captable.ts`

Update the `generateUpdateCapTableChoice` function to:
1. Add `appRewards: Optional AppRewardsConfig` to the choice parameters
2. Generate the conditional marker creation logic
3. Add the `AppRewardsConfig` type definition

## API Usage Example

```daml
-- Without app rewards (backward compatible)
result <- submit issuer do
  exerciseCmd capTableCid UpdateCapTable with
    creates = [OcfCreateStakeholder stakeholderData]
    edits = []
    deletes = []
    appRewards = None

-- With app rewards (new)
result <- submit issuer do
  exerciseCmd capTableCid UpdateCapTable with
    creates = [OcfCreateStakeholder stakeholderData]
    edits = []
    deletes = []
    appRewards = Some AppRewardsConfig with
      couponCount = 5
      featuredAppRight = myFeaturedAppRightCid
```
