# Task: Create Test Helper Functions for Reduced Verbosity

**Date**: 2026-01-21 **Status**: Complete ✅ **Target Repo**: `open-captable-protocol-daml`

## Goal

Create reusable test helper functions to reduce boilerplate in test files and make tests more
focused and readable.

## Problem

Current tests require verbose setup to create prerequisite objects. For example,
`TestStockIssuance.daml` needs 3 separate `UpdateCapTable` calls just to test one issuance:

```daml
testCreateStockIssuance = script do
  TestOcp{...} <- setupTestOcp

  -- Step 1: Create stock class (verbose, 20+ lines)
  result <- submit issuer do
    exerciseCmd cap_table CT.UpdateCapTable with
      creates = [CT.OcfCreateStockClass StockClassOcfData with
        id = "SC_COMMON"
        name = "Common"
        -- ... 10+ more fields
      ]
      edits = []
      deletes = []

  -- Step 2: Create stakeholder (verbose, 15+ lines)
  result2 <- submit issuer do
    exerciseCmd result.updatedCapTableCid CT.UpdateCapTable with
      creates = [CT.OcfCreateStakeholder StakeholderOcfData with ...]
      edits = []
      deletes = []

  -- Step 3: Finally, the actual test (another 20+ lines)
  _ <- submit issuer do
    exerciseCmd result2.updatedCapTableCid CT.UpdateCapTable with ...
```

**Issues:**

- **Repetitive**: Same setup code copied across many test files
- **Hard to read**: Actual test logic buried in setup boilerplate
- **Brittle**: Field changes require updates in many places
- **Verbose**: 60+ lines for a simple test case

## Proposed Solution

Create a `TestHelpers.daml` module with:

1. **Default data factories** — Functions that return valid default OCF data
2. **Setup combinators** — Functions that create common object combinations
3. **Assertion helpers** — Functions for common test assertions

### Example: Improved Test

```daml
testCreateStockIssuance = script do
  TestOcp{issuer, cap_table} <- setupTestOcp

  -- One-liner setup with helpers
  (cap_table', stockClassId, stakeholderId) <- setupForStockIssuance issuer cap_table

  -- Focused test with default data
  _ <- createStockIssuance issuer cap_table' stockClassId stakeholderId
  pure ()
```

### Phase 2: Setup Combinators

Create functions that set up common test scenarios:

```daml
-- Create a stock class and return updated cap table + class ID
setupStockClass : Party -> ContractId CapTable -> Script (ContractId CapTable, Text)
setupStockClass issuer capTable = do
  let classId = "SC_COMMON"
  result <- submit issuer do
    exerciseCmd capTable CT.UpdateCapTable with
      creates = [CT.OcfCreateStockClass (defaultStockClass classId)]
      edits = []
      deletes = []
  pure (result.updatedCapTableCid, classId)

-- Create a stakeholder and return updated cap table + stakeholder ID
setupStakeholder : Party -> ContractId CapTable -> Text -> Script (ContractId CapTable, Text)
setupStakeholder issuer capTable name = do
  let shId = "SH_" <> name
  result <- submit issuer do
    exerciseCmd capTable CT.UpdateCapTable with
      creates = [CT.OcfCreateStakeholder (defaultStakeholder shId name)]
      edits = []
      deletes = []
  pure (result.updatedCapTableCid, shId)

-- Combined: Create stock class + stakeholder for issuance tests
setupForStockIssuance : Party -> ContractId CapTable -> Script (ContractId CapTable, Text, Text)
setupForStockIssuance issuer capTable = do
  (capTable', classId) <- setupStockClass issuer capTable
  (capTable'', stakeholderId) <- setupStakeholder issuer capTable' "Test Holder"
  pure (capTable'', classId, stakeholderId)
```

-- Assert that a batch operation created exactly N objects
assertCreatedCount : Int -> UpdateCapTableResult -> Script ()
assertCreatedCount expected result =
  assertMsg ("Expected " <> show expected <> " created, got " <> show (length result.createdCids))
    (length result.createdCids == expected)

-- Assert that update succeeded and return cap table for chaining
assertUpdateSuccess : UpdateCapTableResult -> Script (ContractId CapTable)
