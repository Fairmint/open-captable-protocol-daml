# DAML Linting Integration

**Date:** 2026-01-13 **Status:** Open

## Summary

Investigate and implement DAML linting/formatting in CI to maintain code quality for DAML smart
contracts.

## Research Findings

### Available Tools

1. **`daml lint`** (Built-in DAML SDK)
   - Part of the DAML SDK, available via `damlc lint`
   - Runs Haskell-style lint checks adapted for DAML
   - Limited documentation on specific rules enabled
   - Open issue [#2410](https://github.com/digital-asset/daml/issues/2410) for `--strict-lint` flag

2. **DAML IDE Extension**
   - Provides real-time linting in VS Code
   - Shows warnings and errors as you type
   - No CLI/CI integration beyond the built-in `daml lint`

3. **No Dedicated DAML Formatter**
   - DAML is based on Haskell syntax
   - Haskell formatters (Ormolu, Fourmolu, Hindent) don't have DAML-specific support
   - Would likely break DAML-specific syntax extensions

### Current State

- No DAML linting in CI
- Code style maintained through manual review and DAML coding standards in `llms.txt`

## Implementation Plan

### Phase 1: Add `daml lint` to CI

Add `daml lint` to the CI workflow after the DAML build step:

```yaml
- name: Lint DAML
  run: |
    # Lint all DAML packages
    for pkg in OpenCapTable-v26 CantonPayments OpenCapTableReports-v01 OpenCapTableProofOfOwnership-v01 Shared Test CouponMinter; do
      echo "Linting $pkg..."
      cd $pkg && daml lint && cd ..
    done
```

### Phase 2: Evaluate Linting Results

1. Run `daml lint` locally on all packages
2. Document any warnings/errors found
3. Decide whether to:
   - Fix existing issues
   - Suppress specific warnings
   - Use `--strict-lint` flag when available

### Phase 3: Consider Custom Formatting Rules

Since no DAML formatter exists:

1. Document coding style in a `.daml-style` guide
2. Consider creating EditorConfig rules for basic formatting (indent, line endings)
3. Explore creating a custom formatter if demand grows

## Acceptance Criteria

- [ ] `daml lint` runs in CI on all DAML packages
- [ ] CI fails if `daml lint` finds errors
- [ ] Document any suppressed warnings with rationale
- [ ] Update `llms.txt` with any new coding standards discovered

## Notes

- DAML is a niche language with limited tooling compared to mainstream languages
- The DAML SDK is actively developed; formatter support may come in future versions
- Monitor [digital-asset/daml](https://github.com/digital-asset/daml) for new linting features
