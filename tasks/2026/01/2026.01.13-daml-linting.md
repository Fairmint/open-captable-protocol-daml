## Research Findings

### Available Tools

1. **`dpm damlc lint`** (Built-in DAML SDK via dpm)
   - Part of the DAML SDK
   - Runs Haskell-style lint checks adapted for DAML (HLint-based)
   - Reports suggestions/warnings at `DsInfo` severity level

2. **DAML IDE Extension**
   - Provides real-time linting in VS Code
   - No CLI/CI integration beyond the built-in lint

3. **No Dedicated DAML Formatter**
   - DAML is based on Haskell syntax
   - Haskell formatters don't have DAML-specific support

## Coding Patterns Discovered

The linter prefers these patterns:

| Instead of                             | Use                        |
| -------------------------------------- | -------------------------- |
| `x == None`                            | `isNone x`                 |
| `x /= None`                            | `isSome x`                 |
| `notElem x xs`                         | ``x `notElem` xs`` (infix) |
| `all (/= "") xs`                       | `"" \`notElem\` xs`        |
| `case x of None -> def; Some v -> v`   | `fromOptional def x`       |
| `case x of None -> def; Some v -> f v` | `optional def f x`         |
| `if x then True else y`                | `x \|\| y`                 |
| `\x -> x /= ""`                        | `(/= "")`                  |
