## Deployment Commands

Run each step below (copy one command at a time):

> **Note**: `upload-dar` now auto-backs up DARs before uploading. No separate backup step needed.

```bash
# Step 0: Build all packages
npm run build

# Step 1: Upload OCP DAR (auto-backs up if not already)
npm run upload-dar -- --package ocp --network devnet
npm run upload-dar -- --package ocp --network mainnet

# Step 2: Upload CouponMinter DAR (auto-backs up if not already)
npm run upload-dar -- --package couponMinter --network devnet
npm run upload-dar -- --package couponMinter --network mainnet

npx tsx scripts/create-couponMinter.ts --network devnet

npm run verify-dars
```

## Notes

- **Auto-backup**: The `upload-dar` script automatically backs up DARs before uploading. This ensures backups are never missed and the exact deployed DAR is always preserved.
- **Dynamic template IDs**: Scripts now compute template IDs from `packages.ts` which reads versions from `daml.yaml`. This ensures the correct package version is always used after upgrades (single source of truth).
- CouponMinter uses consuming choices (`MintCoupons`), so the contract ID changes on every call. Clients must query for the current contract rather than using a stored contract ID.
- The `intellect` party is used as the operator for CouponMinter.
- Both `intellect` and `5n` providers receive the DAR uploads.
