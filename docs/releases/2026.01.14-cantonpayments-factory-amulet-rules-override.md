# Release: CantonPayments v0.0.32 - Factory AmuletRules Override

**Date**: 2026-01-14
**Author**: NC
**Networks**: devnet, mainnet
**Type**: Minor version upgrade (backwards compatible)

## Summary

Exposes the `amuletRulesCidOverride` functionality in `Factory_ExecuteBatchTransfers` choice. When `Some`, uses the provided AmuletRules contract ID instead of the stored `amuletRulesCid` for all transfer operations.

## What's Changed

| Package | Old Version | New Version | Change Type |
|---------|-------------|-------------|-------------|
| **CantonPayments** | 0.0.31 | 0.0.32 | Minor (backwards compatible) |

### Factory_ExecuteBatchTransfers Choice

New optional parameter added at the end:

```daml
optAmuletRulesCidOverride : Optional (ContractId AmuletRules)
```

- When `None`: Uses stored `amuletRulesCid` (existing behavior)
- When `Some`: Uses the provided AmuletRules contract ID for:
  - All `PersonalAirdrop_ExecuteTransfer` calls
  - The re-lock transfer operation (when using LockedAmulet)

## Backwards Compatibility

This change is backwards compatible because:

1. The new parameter is `Optional` - existing callers passing `None`/`null` get current behavior
2. No changes to template fields or signatories
3. No changes to return types
4. Parameter is added at the end of the choice
